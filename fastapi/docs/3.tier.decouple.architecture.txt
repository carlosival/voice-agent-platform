Detailed Internal Architecture per Worker NodeInstead of running a single monolithic process, each Tier 3 server instance runs a resource-isolated pool structure:The Signaling Gate: The client opens a WebSocket connection directly to the assigned worker machine. The worker grabs the query token, hits Redis to verify and instantly delete it (GETDEL valid_tokens:token), ensuring it can never be used again.Allocating the Lifecycle Worker: Once authorized, the worker initializes your PeerSession and spins up the asynchronous PipelineWorker loop we designed earlier to isolate client inputs.WebRTC Media Negotiation: The client and worker exchange WebRTC SDP Offers/Answers over the WebSocket. Once completed, the client establishes a peer-to-peer UDP connection over a high-range port directly to the worker. import sys
import asyncio

async def handle_sigterm(worker_id, redis_client):
    logger.info("SIGTERM received! Entering graceful draining mode...")
    
    # 1. Update status in Redis so Tier 1 stops sending new calls here
    await redis_client.hset(f"workers:{worker_id}", "status", "draining")
    
    # 2. Continually monitor active calls in your local active_connections set
    while len(active_connections) > 0:
        logger.info(f"Draining... {len(active_connections)} calls still active.")
        await asyncio.sleep(5)
        
    # 3. Once empty, close background loops and exit cleanly
    logger.info("All calls finished. Safe to shut down container.")
    await close_all_peers()
    sys.exit(0)
